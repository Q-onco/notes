# Session Log — 2026-06-04 · Session 1

## Fix: DOI resolver cross-browser bug

### Problem
DOI search popover worked in Chrome but not Firefox/Safari.

### Root cause
The dismiss-backdrop was `position: fixed; inset: 0; z-index: 30` placed inside a
`position: relative` container (`.doi-wrap`). Chrome tolerates this stacking;
Firefox/Safari pull the fixed element into the viewport stacking context and it
covered the popover's input and Resolve button, making them unclickable.

### Fix
- Removed `.doi-backdrop` div entirely
- Added a `$effect` that registers a capture-phase `window` click listener when
  `doiOpen` is true; closes popover when click target is outside `.doi-wrap`
- Removed dead `.doi-backdrop` CSS rule
