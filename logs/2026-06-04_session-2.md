# Session Log — 2026-06-04 · Session 2

## Summary
Multi-user support added — each GitHub login now gets its own encrypted namespace in the repo. Legacy data migrates automatically on first login.

## Problem Solved
Single shared flat paths (`data/notes.enc`) meant any second user would overwrite the first user's data. The encryption (PAT = key) already provided isolation, but the storage paths did not.

## What Changed

### `app/src/lib/github.ts`
- Replaced `export const PATHS` with `export function makePaths(username: string)` — returns paths prefixed as `data/${username}/notes.enc`, `settings/${username}/keys.enc`, etc.
- Added `export const LEGACY_PATHS` — the old flat paths, used only for migration fallback
- Added `export async function loadEncFileFallback<T>(...)` — tries the user-scoped path first; if not found (sha === null), falls back to the legacy flat path and returns `sha: null` so the next save creates the new user-scoped file

### `app/src/lib/store.svelte.ts`
- Updated import: `makePaths, LEGACY_PATHS, loadEncFileFallback` instead of `PATHS`
- Added `get paths()` getter: `return makePaths(this.userLogin)` — all path access goes through here
- `loadAll()`: replaced 26 `loadEncFile(tok, PATHS.xxx, ...)` calls with `loadEncFileFallback(tok, p.xxx, L.xxx, ...)` using a local `load` alias for brevity
- All `saveXxx()` methods: replaced `PATHS.xxx` with `this.paths.xxx` (global replace)

## Migration Behaviour
- User logs in for the first time after this update
- `loadAll()` checks `data/${username}/notes.enc` → doesn't exist → falls back to `data/notes.enc` → found → loads data with `sha: null`
- User edits and saves → data written to `data/${username}/notes.enc` as a new file
- Old flat files remain in the repo but are never read again for this user
- A second user logs in → their own `data/${username2}/notes.enc` namespace, zero overlap
