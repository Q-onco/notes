# 2026-05-20 Plan 1 — Enzo Virtual Pet System

## Goal
Make Enzo (the golden shepherd dog) a living ambient presence throughout the app — emotional state driven by real activity, dual-timezone aware, zero impact on chat pipeline.

## Files
- [ ] `app/src/lib/enzo-engine.ts` — dual-TZ utils, 6 dimensions, 40 archetypes, ~700 reaction phrases
- [ ] `app/src/components/EnzoDog.svelte` — golden shepherd SVG with 5 visual emotion families
- [ ] `app/src/components/Shell.svelte` — den button replacing enzo-toggle, 3-min pet state polling
- [ ] `app/src/components/Enzo.svelte` — emotion prop, dog replaces "E" avatar

## Architecture
- `buildEnzoSnapshot(store)` → plain data snapshot (no Svelte reactivity)
- `getEnzoState(snapshot)` → `{ archetype, emotion, visual, phrase }` (pure computation)
- Shell polls every 3 min with `untrack` so no reactive subscriptions
- EnzoDog.svelte: CSS custom properties on SVG inherit to `.tail`/`.ear-l`/`.ear-r`
- 5 visual families: `bright | neutral | sleepy | worried | sad`
- 40 archetypes, priority-ordered (most constrained first)
- `opening` prop triggers stretch CSS animation on den click

## Dual timezone rule
BOTH `Asia/Kolkata` and `Europe/Berlin` are checked simultaneously for ALL time triggers.

## No-touch zones
- `send()` in Enzo.svelte (LLM pipeline)
- streaming state
- chat session logic
