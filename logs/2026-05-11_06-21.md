## Bugs fixed (commit b3f252c)

### 1. `onError` not destructured — RichEditor.svelte
`onError` was declared in the TypeScript props type but omitted from the
`let { ... } = $props()` destructuring. Optional chaining (`onError?.()`)
does NOT guard against `ReferenceError` for undeclared variables — it only
handles null/undefined mid-chain. Three call sites were affected:
- Line 356: catch block after image upload failure → silent crash
- Line 369: catch block after file upload failure → silent crash
- Line 449: after successful replace-all (success count notification) → crash

**Fix:** Added `onError` to the destructuring pattern.

### 2. Text alignment buttons broken — RichEditor.svelte
`cmd('setTextAlign', { alignment: 'left' })` passed an object as the first
argument to Tiptap's `setTextAlign(alignment: string)`. Tiptap expects a
plain string. All three alignment buttons (left/center/right) silently did
nothing.

**Fix:** Replaced all three buttons with direct `editor?.chain().focus().setTextAlign('left').run()` calls, and updated `isActive` checks to `editor?.isActive({ textAlign: 'left' })` (Tiptap supports attribute-object overload of isActive).

### 3. Enzo analysis rendered raw markdown — Editor.svelte
`{analyseResult}` rendered the Groq/Llama response as plain text, so
`**bold**`, `- list items`, `### headers` appeared as literal characters.

**Fix:** Added `import { marked } from 'marked'`; changed template to
`{@html marked.parse(analyseResult)}`. Added scoped `:global(.analysis-body ...)` CSS for `p`, `ul`, `ol`, `li`, `strong`, headings to match the dark clinical theme.
