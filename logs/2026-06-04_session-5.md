## Enzo Chat Improvements

### Context limits raised (all grotesquely conservative for 128k token model)
| Location | Before | After |
|---|---|---|
| Note body | 2,000 chars | 40,000 chars |
| Files viewer text | 8,000 chars | 40,000 chars |
| PDF extracted | 12,000 chars | 60,000 chars |
| Text/code file | 8,000 chars | 40,000 chars |

### Message history window: 10 → 30

### New Chat (+) button
- Archives current session under timestamp ID (preserves in history)
- Opens clean session; exits resume mode if active

### History: session lookup + Resume
- Fixed: history was looking up sessions by `date` — broke when multiple sessions shared a date (after startNewChat)
- Changed to look up by `id`
- Added **Resume →** button in history detail view
- `resumedSessionId` state loads historical session into chat
- Resumed session banner with "← Back to today" link

### File auto-fetch on mention
- Scans user message for filenames matching `store.files`
- Handles both R2 storage (`r2Key`) and legacy base64 (`data`) — base64 fix was critical (older files)
- `extractPdfText` extracted to shared `lib/pdfUtils.ts`

### File context persistence
- `sessionFileContexts: {name, content}[]` state (up to 3 files)
- Saved to `ChatSession.fileContexts` in encrypted store on first fetch
- Restored via `$effect` on session change — survives page refresh + resume
- Per-file chips in context bar with individual × clear

### Loading indicator during file extraction
- `fetchingFile` state shows "Reading <filename>…" with animated dots
- Error toast if PDF is scanned/image-only or extraction fails (was silent)

### Multiple files in context (up to 3)
- Each session can hold up to 3 files simultaneously
- All file contents joined and passed as `## File: <name>` blocks to Enzo
- Each has its own context bar chip

### Copy button on messages
- Clipboard icon appears on hover on every message
- `navigator.clipboard.writeText(msg.content)`

### Regenerate button
- ↺ icon on last assistant message only, hidden while streaming
- Drops last assistant message, re-sends last user message

### Markdown rendering
- `marked` v14 (already in project) now used to render Enzo responses
- `{@html renderEnzo(msg.content)}` with `.enzo-md` CSS
- Full support: headers, bold, italic, lists, tables, code blocks, blockquotes, links
- Links open in `_blank` via post-process regex (avoids marked v14 renderer API change)

### Syntax highlighting
- `marked-highlight` + `highlight.js` installed
- Languages: R, Python, bash/shell, TypeScript, JavaScript, SQL
- GitHub-style token colours; dark mode override via `prefers-color-scheme`

### System prompt update
- Removed "No emojis" restriction — Enzo can use emojis sparingly
- Added markdown + URL formatting instructions
- Enzo now structures responses with headers, lists, tables where appropriate
- Includes clickable URLs/links to sources when relevant
