## Code change: separate API auth from encryption key
**Problem:** The app used a single PAT for two unrelated purposes:
1. GitHub API authentication (reading/writing .enc files)
2. PBKDF2 encryption passphrase for all .enc file contents

When GitHub revoked the PAT, both broke. The fix separates them permanently.

**Changes:**
- `app/src/lib/github.ts`: GH_API_TOKEN constant reads from VITE_GH_API_TOKEN build-time env var; headers() uses this constant; validateToken() uses embedded token for API auth
- `app/src/components/Login.svelte`: label/placeholder updated to "Access password"
- `app/.env.local` (gitignored): contains the new API token for builds — NEVER commit this file

**Result:**
- Login field is now purely an encryption passphrase — never touches the GitHub API
- GitHub API token rotates independently without affecting encryption
- Dr. Amritha logs in with her existing passphrase; data decrypts as before

## Rule reinforced
Never write any credential, token, or key value in a session log or any committed file.
All credentials live only in VPS .env.local files (gitignored) and secure memory only.
