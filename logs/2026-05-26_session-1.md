### A3 ‚Äî Jobs.svelte + groq.ts: source badges, filter chips, AI match score

**groq.ts ‚Äî `scoreJobMatch()` added:**
- Takes: job title, company, description, CV summary string
- Uses `MODELS.quick` (fast, 200 token response)
- Returns `{ score: number (0‚Äì100); rationale: string }` via JSON extraction
- System prompt: realistic scoring ‚Äî 80+ = strong match, 60‚Äì79 = good, 40‚Äì59 = partial, <40 = weak

**Jobs.svelte changes:**
- `scoreJobMatch` imported from groq.ts
- New state: `sourceFilter` (string | 'all'), `matchScores` (Record<id, {score, rationale}>), `matchScoring` (Record<id, bool>)
- `displayFeed` derived updated to filter by `sourceFilter` in addition to region/type
- `feedSources` derived: unique sources from current feed (drives chip list)
- `runMatchScore(job)` function: builds CV summary from `store.profile` + `store.cvProfile.experience`, calls `scoreJobMatch()`, caches result in `matchScores`
- `matchScoreClass(score)` ‚Üí CSS class: match-high / match-good / match-partial / match-low
- `sourceClass(source)` ‚Üí CSS class: src-jobvector / src-nature / src-euraxess / src-other

**UI additions:**
- Source filter chips row (shown only when >1 source in feed): All / JobVector / Nature Careers / Euraxess ‚Äî colour-coded, active state per source colour
- Per job card: `job-source-badge` (colour-coded pill replacing plain italic source text)
- Per job card: `.enzo-match-btn` ‚Äî "Match score" button ‚Üí fires `runMatchScore`; shows spinner while scoring; disabled + "Scored" once cached
- Per job card: `.match-result` div ‚Äî coloured badge (Match 84%) + 2-sentence rationale shown inline below description
- Per job card: "Save + Letter" one-click button ‚Äî calls `saveJobToTracker()` then navigates to cover letter tab with job prefilled

**CSS added:** source badges (`.src-jobvector`, `.src-nature`, `.src-euraxess`), source chips, match badge colours (`.match-high/good/partial/low`), match rationale text
