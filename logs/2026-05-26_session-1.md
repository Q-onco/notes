### A3 — Jobs.svelte + groq.ts: source badges, filter chips, AI match score

**groq.ts — `scoreJobMatch()` added:**
- Takes: job title, company, description, CV summary string
- Uses `MODELS.quick` (fast, 200 token response)
- Returns `{ score: number (0–100); rationale: string }` via JSON extraction
- System prompt: realistic scoring — 80+ = strong match, 60–79 = good, 40–59 = partial, <40 = weak

**Jobs.svelte changes:**
- `scoreJobMatch` imported from groq.ts
- New state: `sourceFilter` (string | 'all'), `matchScores` (Record<id, {score, rationale}>), `matchScoring` (Record<id, bool>)
- `displayFeed` derived updated to filter by `sourceFilter` in addition to region/type
- `feedSources` derived: unique sources from current feed (drives chip list)
- `runMatchScore(job)` function: builds CV summary from `store.profile` + `store.cvProfile.experience`, calls `scoreJobMatch()`, caches result in `matchScores`
- `matchScoreClass(score)` → CSS class: match-high / match-good / match-partial / match-low
- `sourceClass(source)` → CSS class: src-jobvector / src-nature / src-euraxess / src-other

**UI additions:**
- Source filter chips row (shown only when >1 source in feed): All / JobVector / Nature Careers / Euraxess — colour-coded, active state per source colour
- Per job card: `job-source-badge` (colour-coded pill replacing plain italic source text)
- Per job card: `.enzo-match-btn` — "Match score" button → fires `runMatchScore`; shows spinner while scoring; disabled + "Scored" once cached
- Per job card: `.match-result` div — coloured badge (Match 84%) + 2-sentence rationale shown inline below description
- Per job card: "Save + Letter" one-click button — calls `saveJobToTracker()` then navigates to cover letter tab with job prefilled

**CSS added:** source badges (`.src-jobvector`, `.src-nature`, `.src-euraxess`), source chips, match badge colours (`.match-high/good/partial/low`), match rationale text

## Deploy C — Visual Paper Compare

### C1 — groq.ts: `comparePapersStructured()`
New export type `CompareStructured`:
```typescript
{ scores: Record<string, number[]>; axes: string[]; table: { dimension: string; [k: string]: string }[]; verdict: string }
```
- Uses `MODELS.enzo` with 1200 tokens
- Returns structured JSON: 6-axis scores (Sample Size, Methodology, Novelty, HGSOC Relevance, Statistical Power, Translatability), comparison table (one sentence per paper per dimension), 2-sentence verdict
- Scores normalised to 1–5 range with fallback to 3

### C2 — Research.svelte: Full-screen modal with SVG radar
Replaced old streaming compare card with a `position: fixed` full-screen modal:
- **Papers strip**: P1/P2/P3 coloured pills (blue/purple/green) with title + journal + year
- **SVG radar chart**: viewBox 520×400, 6 axes at 60° intervals, 5 grid rings, two overlapping polygons (P1 blue, P2 purple) with fade-in animation, score dots on each vertex, axis labels, numeric tick marks
- **Comparison table**: per-axis per-paper sentences; dimension column shows inline score badges (P1/P2 coloured)
- **Verdict strip**: highlighted bottom bar with Enzo's 2-sentence verdict
- **Save as note**: exports structured data as markdown note with radar scores + comparison table
- Escape key / click-outside to close

**State changes:**
- Removed `compareResult`, `compareStreaming`, `showCompare`
- Added `compareStructured: CompareStructured | null`, `compareModalOpen`, `compareModalLoading`
